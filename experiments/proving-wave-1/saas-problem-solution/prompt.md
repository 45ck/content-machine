# Experiment Brief

Produce a vertical creator-style SaaS short that follows the
`hook -> problem -> solution -> proof/demo -> CTA` shape.

Constraints:

- Keep all generated artifacts under this lane folder.
- Prefer the repo's built-in skills and harness scripts.
- Aim for one real, reviewable MP4 rather than a mock placeholder.
- Do not assume hidden product assets exist.
- If no real product is supplied, use a fictional SaaS and state that
  clearly in the report.

Chosen proving concept:

`CallScribe`, a fictional SaaS that turns sales calls into CRM notes and
follow-up tasks automatically.

Reasoning:

- It matches the rollout doc's suggested angle:
  `Stop turning sales calls into CRM notes by hand.`
- The pain point is concrete enough for a problem-solution promo.
- It avoids unsupported claims about a real vendor.
- It can still exercise the repo's script, audio, visuals, render, and
  review stack.

Known limitation:

- Without supplied screenshots or dashboard recordings, the visuals path
  uses generic local repo motion clips rather than true product demo
  assets.
