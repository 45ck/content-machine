# SaaS Problem Solution

Status: `showcase candidate`

This is the current commercial/product showcase candidate. It uses
product-style proof cards, direct pain/solution copy, captions, and a
CTA. It is useful as a capability example, but it is weaker than the
golden Reddit story because the proof cards still feel more like a
motion deck than a native UGC ad.

Use this lane for product or software shorts that should feel like a
creator ad instead of a generic explainer.

Core shape:

- pain or promise in the first line
- product appears as the answer, not the setup
- proof or demo beat before CTA
- avatar, UGC, or faceless support can all work
- proof cards need visible state changes, zooms, cursor motion, or
  transition pulses to avoid brochure/slideshow energy
