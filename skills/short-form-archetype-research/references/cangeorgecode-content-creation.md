## Archetype

`ugc-avatar-product-short`

## Production Pattern

Script JSON plus user-provided b-roll and optional avatar assets become a
batch of platform-ready short-form videos.

## Useful Extraction

- `swipe_files` script input folder
- `brolls`, `media`, and `output` folder contract
- optional avatar input
- HeyGen dependency boundary
- sample approval before full batch
- positioning as a repeatable production factory

## Content-Machine Implication

UGC/avatar jobs need stricter input contracts than faceless explainers:
product proof, brand assets, presenter/avatar provenance, b-roll source, and
sample approval should be required before batch generation.

## Asset Policy

Do not reuse demo GIFs, avatars, or videos. Use the repo to define folder
contracts and approval gates.
